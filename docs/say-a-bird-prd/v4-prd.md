## 1. 项目概述

### 1.1 一句话描述

> 一款通过情绪日记生成鸟类插画、积累个人专属"情绪天空"的微信小程序——你的自然系情绪树洞。

### 1.2 项目背景

**为什么要做**：

当代年轻人面临日益增长的情绪压力，但传统的情绪日记产品存在两个核心问题：一是"写日记"门槛高、难以坚持；二是纯文字记录缺乏情感反馈，用户感受不到被倾听。市场上已有的情绪可视化产品（如 Mood Chonk 的情绪小怪兽、MoodFlora 的情绪花园）验证了"情绪→视觉化生物"这一模式的用户需求，但尚无产品将鸟类这一自然元素与情绪记录结合，更没有"情绪天空"这种具有时间积累感的可视化形式。

同时，中国观鸟群体正在快速增长，观鸟社群活跃度高但缺乏数字化的情感连接工具。这为产品提供了精准的冷启动渠道和天然的种子用户群。

**当前现状**：

用户目前的情绪记录方式主要有：手动写日记（门槛高、难坚持）、使用通用情绪追踪 App（如 Daylio、Moodistory，功能实用但缺乏情感温度）、在社交平台发泄（缺乏隐私性）、或者干脆不记录。市场上"情绪+可视化生物"赛道的产品（Mood Chonk、Voidpet Garden）均为海外产品，中国市场存在空白。

**核心假设**：

1. 用户愿意通过"说话/打字/一键放空"的低门槛方式记录情绪（而非手动选择情绪标签）
2. "情绪→本土高网感鸟类"的映射能为鸟类爱好者和年轻群体提供足够的情感共鸣、社交梗和收集动力
3. "情绪天空"的视觉积累与提取无害词云的分享效果具有足够的社交传播力（小红书/朋友圈分享）
4. 观鸟社群可以作为有效的冷启动渠道

### 1.3 项目目标

| 目标类型 | 描述 | 衡量指标 | 目标值 |
|---------|------|---------|--------|
| 核心目标 | 验证"情绪日记→鸟类生成"核心玩法的用户接受度 | 次日留存率 | MVP阶段 ≥ 30% |
| 核心目标 | 建立种子用户群 | 月活跃用户数（MAU） | 上线3个月 ≥ 5,000 |
| 次要目标 | 验证商业化模型可行性 | 付费转化率 | ≥ 5% |

| 非目标 | 不做心理咨询/诊断功能 | — | — |
| 非目标 | 不做实时社交聊天功能 | — | — |

## 2. 目标用户

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| 核心用户群 | 两类并行：(A) 18-35岁有情绪记录需求的年轻人，追求治愈感和反内耗；(B) 观鸟爱好者，对鸟类有天然认同感 |
| 用户规模估算 | 中国观鸟爱好者约200-500万；泛自然/治愈兴趣年轻人群体数千万级 |
| 使用频率 | 日活场景：每天1-3次情绪记录；周活场景：回顾情绪天空、查看图鉴、分享 |
| 技术水平 | 微信小程序熟练用户，无需额外学习成本 |
| 核心诉求 | (A) 极低门槛的情绪出口 + 有温度的情感反馈 + 社交表达的“嘴替”；(B) 鸟类知识融入日常 + 独特的鸟类收集体验 |

### 2.2 用户分层

- **P0 用户A**：有情绪记录习惯或意愿的年轻人（18-35岁），追求治愈感和仪式感 — 必须满足
- **P0 用户B**：观鸟爱好者，对鸟类品种有认知，会关注鸟类映射的本土准确性和趣味性 — 必须满足（冷启动核心用户）
- **P1 用户**：纯粹被视觉效果吸引的泛用户，可能从小红书/朋友圈看到词云天空图分享后进入 — 应该满足

## 3. 核心场景

### 场景 0：首次使用引导（Onboarding）

- **触发条件**：用户首次打开小程序
- **用户行为**：
  1. 展示欢迎页：一句话介绍产品概念（"说出你的心情，让它变成一只鸟"）+ 治愈风格的插画
  2. 引导用户完成第一次情绪记录（预设引导文案："今天过得怎么样？随便说说吧"）
  3. 系统识别情绪，展示第一只鸟的生成过程（放慢动画，增加仪式感）
  4. 鸟类飞入空白天空，配合文字说明："这是你的第一只鸟——{鸟类名}，它代表你此刻的{情绪}"
  5. 简要提示：天空会随时间积累更多鸟类，每月天空会保留一只守护鸟，其余会迁徙到记忆森林
- **期望结果**：用户在30秒内理解产品玩法，完成第一次记录，感受到"哇"的惊喜
- **异常处理**：用户跳过引导时，首页天空显示一只示例鸟（半透明）+ 引导气泡

### 场景 1：日常情绪倾诉（含“失语”状态）

- **触发条件**：用户在一天中经历了某种情绪波动（开心、烦躁、焦虑、感动等）想要倾诉，或者极度疲惫处于什么都不想说的“失语”状态
- **用户行为**：
  1. 打开小程序，进入首页（当前天空画布）
  2. 点击"说只鸟"按钮，进入输入界面
  3. **[常规倾诉]**：通过文字输入或语音输入，自由描述今天发生的事和感受
  4. **[一键放空]**：如果用户极度疲惫不想说话，可长按输入区旁的“深呼吸”按钮，系统直接记录一次“放空/疲惫”状态
  5. 系统自动调用大模型识别复合情绪，展示匹配的鸟类预览（品种+姿势+色调）
  6. 用户可选择确认或手动调整情绪标签
  7. 确认后，鸟类以动画形式飞入天空画布
- **期望结果**：用户感受到"被倾听"，看到自己的情绪化身为一只具体的鸟。无论有强烈的倾诉欲还是只想叹气，都能得到接纳。记录次数不设商业限制。
- **异常处理**：情绪识别不准确时，允许手动调整；网络异常时内容本地缓存，界面显示"等待网络连接后生成你的鸟"占位状态。

### 场景 2：回顾情绪天空与记忆森林（守护鸟机制）

- **触发条件**：用户想回顾自己的情绪历程，或者单纯想看看自己的天空
- **用户行为**：
  1. 打开小程序，首页即为当月天空画布（背景光照随真实时间变化，装饰元素随鸟类数量进化）
  2. 默认显示近7天的鸟类，可切换为"本月全部"
  3. 点击天空中的任意鸟类，弹出该条日记的摘要和情绪标签
  4. 点击展开可查看完整日记内容，支持编辑和删除
  5. 切换到"记忆森林"入口，浏览往月迁徙的鸟类（按月份组织的森林）。**当月天空中会永久保留上个月的 1 只代表性“守护鸟”**，避免天空被彻底清空带来的剥夺感。
  6. 在记忆森林中同样可以点击鸟类查看历史日记
- **期望结果**：用户获得情绪回顾的满足感，感受到时间积累带来的成就感，且免于资产被清空的失落。

### 场景 3：分享情绪天空（隐私保护词云版）

- **触发条件**：用户觉得自己的天空很好看，想在社交平台展示情绪氛围，但不想暴露日记内容
- **用户行为**：
  1. 在天空画布页面点击"分享"按钮
  2. 系统生成精美的天空截图，包含：**天空背景 + 所有鸟类 + 从日记中提取的3-5个“无害关键词”（如：好天气、咖啡、有点累，以极简手写体散落在云层边形成词云） + 小程序码**，绝不包含完整日记内容。
  3. 用户选择分享到朋友圈、小红书、微信好友或保存到相册
- **期望结果**：生成的图片兼具视觉美感和充足的“情绪信息熵”，能在社交平台引发共鸣和好奇；其他人扫码可直接进入小程序
- **异常处理**：鸟类过少时（<3只），提示用户"再记录几次情绪，天空会更丰富哦"

### 场景 4：鸟类图鉴收集

- **触发条件**：用户想查看自己已解锁的鸟类品种，或者好奇还有哪些鸟类未解锁
- **用户行为**：
  1. 进入"图鉴"页面，查看已解锁和未解锁的鸟类
  2. 已解锁的鸟类展示品种名称、鸟类小知识、对应情绪、首次出现日期
  3. 未解锁的鸟类显示为剪影+解锁提示（如"当你感到极度治愈时，它会出现"）
  4. 季节限定鸟类有特殊标识和倒计时
- **期望结果**：激发用户的收集欲望，增加使用频率和留存；观鸟爱好者能从图鉴中获得本土鸟类知识的满足感

### 4.2 功能详述

#### 功能：情绪输入与识别

- **目的**：让用户以最低门槛倾诉情绪，系统自动理解并给出反馈
- **输入**：用户的文字、语音，或“一键放空”长按指令
- **处理逻辑（LLM 驱动）**：
  1. 废弃传统 NLP，全面接入轻量级大语言模型 API（如 DeepSeek-Chat 或 Kimi）。
  2. 使用系统 Prompt 约束输出：`你是一个心理分析师，阅读以下日记，输出 JSON 格式：{"emotion_type": "核心情绪", "intensity": 1-5, "posture": "姿势", "color_tone": "色调", "keywords": ["无害词1", "无害词2"]}`
  3. 从日记中提取不暴露隐私的关键词（keywords）用于后续生成分享图的词云。
- **商业与体验边界条件**：
  - **核心红线**：情绪记录**永远免费，不限次数**，严禁在记录流程中弹出广告。
  - **单日防刷上限**：50次/天（针对所有用户，用于防机器刷量，对正常用户无感）。触达上限时系统温柔提示："今天已经说了很多了，天空也需要休息，明天继续吧。"
  - 敏感内容检测：触发自伤词汇时展示关怀提示及心理援助热线（400-161-9995）。

#### 功能：情绪→鸟类映射引擎

- **目的**：将抽象的情绪转化为具体的本土鸟类形象，建立情感连接
- **处理逻辑**：三维映射模型
  1. **品种维度**（MVP 35种）：精选中国本土、自带网络属性的鸟类（详见 10.4 附录），每种鸟类对应一种核心情绪隐喻。
  2. **姿势维度**（每种3-4种）：如展翅（释放）、栖息（平静）、缩头（退缩/疲惫）
  3. **色调维度**（每种3-4种）：暖色调、冷色调、柔和色调、鲜艳色调
- **素材规模**：MVP 35种×3姿势×3色调=315张图片（采用统一的极简线段治愈风插画）。
- **鸟类知识融入**：每只鸟生成时附带一句该鸟类的科普小知识（如"你知道吗？夜鹭白天主要负责思考人生"），满足观鸟爱好者的知识需求。

#### 功能：天空画布（实时光照+进化装饰 双层系统）

- **目的**：为用户提供沉浸式的情绪可视化空间
- **双层架构**：
  - **底层：实时光照**（根据本地时间变化）：06:00-08:00清晨（橙粉），08:00-17:00白天（明蓝），17:00-19:00黄昏（暖金），19:00-06:00夜晚（深蓝）。
  - **上层：进化装饰**（根据当月鸟类数量叠加）：1-5只（基础云朵）→ 6-15只（山峦）→ 16-30只（彩虹/星座）→ 30只以上（流星雨/极光）。

#### 功能：记忆森林与守护鸟迁徙

- **目的**：解决天空鸟类过多的问题，同时创造有温度的情绪归档体验，消除剥夺感。
- **处理逻辑**：
  1. 每月最后一天 23:59 触发迁徙机制。
  2. 系统根据本月词云或最高频情绪，自动选出 **1 只代表性的鸟类作为“月度守护者”，永久保留在次月的天空中**。
  3. 其余鸟类伴随温柔的迁徙动效归档至“记忆森林”。森林以月份为单位组织，树林丰茂度与该月鸟类数量正相关。

## 5. 竞品分析

### 5.1 竞品概览

| 竞品 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| Mood Chonk | 情绪→小怪兽收集 | 角色设计可爱，收集机制成熟 | 仅iOS，无中文，角色与自然无关 |
| Voidpet Garden | 情绪→魔法宠物花园 | 花园养成感强，心理健康定位专业 | 风格偏奇幻非自然系 |
| MoodFlora | 情绪→植物可视化 | 界面极简美观，操作门槛极低 | 功能单一，缺乏深度互动 |

### 5.2 差异化策略

**我们的独特价值**：
1. **本土网感鸟类定位**：放弃传统的“动物园”生物（企鹅、火烈鸟等），选用在中国城市常见、且在社交网络上自带“人设”的鸟类（如北长尾山雀、珠颈斑鸠），极大增强共鸣感。
2. **绝对克制的商业化**：不同于竞品限制每日记录次数的做法，本产品坚持记录自由，只对报告、皮肤等外围增值服务收费。
3. **天空+词云双层叙事**：解决了“纯天空图太单调，带日记又没隐私”的分享痛点，提取无害词形成词云，打造社交货币。
4. **一键放空与守护鸟**：体察年轻人的情绪死角，允许无字记录；月度迁徙不“一刀切”，留下情感牵绊。

## 9. 技术约束与风险

### 9.1 技术约束
- **平台约束**：微信小程序原生开发（避免跨端框架引发 AI 代码幻觉）。云开发（CloudBase）实现无服务器架构。
- **大模型接入**：云函数对接外部 LLM API（如 DeepSeek/Kimi），替代传统 NLP。
- **性能优化**：画布渲染需使用 Canvas 2D API，默认近7天视图控制同屏渲染数量防卡顿。
- **合规要求**：不涉及心理诊断；数据加密存储；个体工商户主体注册保障后续商业化能力。

### 9.2 风险评估
| 风险 | 缓解措施 |
|------|---------|
| 插画素材工作量大 | MVP优先落实 35种核心鸟类。使用 AI 绘图工具统一“极简治愈线条风”批量生成。 |
| 复合情绪提取不准 | 依靠 LLM 强大的语境理解能力，并通过 Prompt 持续调优，保留手动修改作为兜底。 |
| 画布拥挤性能下降 | 设计“月度迁徙”与“默认近7天视图”，合理控制同屏元素。 |
| 商业化测试受阻 | 采用“开发环境(个人主体)与生产环境(个体户)”分离策略，零成本无缝切换。 |
