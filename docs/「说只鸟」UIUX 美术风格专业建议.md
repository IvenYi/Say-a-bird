## 「说只鸟」UI/UX 美术风格专业建议

### 整体视觉定位

**核心关键词**：自然系、治愈、克制、呼吸感

这款产品需要在"足够美但不过度设计"之间找到平衡。用户来这里是为了倾诉情绪，不是为了被视觉轰炸。美术风格应该像一个安静的树洞，而不是热闹的游乐场。

------

### 1. 天空画布的美术风格

**背景设计**：

- **风格**：水彩渐变 + 极简云层，避免过于写实或卡通
- **色彩饱和度**：中低饱和度（饱和度 40-60%），避免过于鲜艳刺眼
- **层次**：2-3 层渐变即可，不要做复杂的云朵细节
- **参考方向**：宫崎骏动画的天空（《千与千寻》《起风了》）、Ghibli 风格的自然场景

**四个时段的色彩方案**：

- **清晨（6-8点）**：橙粉 → 浅蓝渐变，色温偏暖，饱和度 50%
- **白天（8-17点）**：天蓝 → 白色渐变，明度最高，给人开阔感
- **黄昏（17-19点）**：暖金 → 橙红渐变，最有情绪感染力的时段
- **夜晚（19-6点）**：深蓝 → 紫黑渐变，低明度但不压抑，保留星光点缀

**三栖基底**：

- **风格**：极简线条插画，类似 Moleskin 笔记本的手绘风
- **颜色**：单色或双色，避免彩色（如深棕色线条 + 浅蓝水面）
- **细节度**：能看出是树枝/水面/滩涂即可，不要画树叶、波纹等细节

------

### 2. 鸟类插画的风格

这是整个产品的视觉核心，必须慎重。

**推荐风格：极简线段治愈风**

**具体特征**：

- **线条**：流畅的单线条勾勒轮廓，线宽 2-3px，不要描边
- **填充**：纯色块填充，保留鸟类真实羽色但降低饱和度（饱和度 30-50%）
- **细节度**：能看出品种特征即可（如长尾山雀的长尾巴、夜鹭的缩头姿势），不要画羽毛纹理
- **眼睛**：简化为圆点或小圆圈，赋予"呆萌感"
- **比例**：Q版但不过度变形，头身比约 1:1.5（比真实鸟类头大一些）

**为什么选择这种风格？**

1. **制作效率高**：AI 绘图工具（如 Midjourney、Stable Diffusion）能快速生成统一风格
2. **情感温度适中**：比照片写实更有亲和力，比卡通风更克制
3. **适配性强**：在不同背景光照下都能保持辨识度
4. **符合观鸟用户审美**：观鸟爱好者对过度卡通化的鸟类会有抵触，极简风既有艺术感又保留真实性

**参考案例**：

- **Duolingo 的吉祥物风格**（但更简化）
- **Monument Valley 的角色设计**（几何化但有温度）
- **日本插画师 Noritake 的作品**（极简线条 + 纯色块）

**AI 绘图 Prompt 参考**：

```
A minimalist line art illustration of a [鸟类名称], 
simple solid color fill with low saturation, 
clean single-line contour, 
cute but not overly cartoonish, 
white background, 
healing style, 
2D flat design
```

### 3. UI 界面风格

**整体原则**：留白 > 填充，克制 > 炫技

**配色方案**：

- **主色**：天空蓝（#87CEEB）或自然绿（#A8D5BA），低饱和度
- **辅助色**：暖灰色（#D3D3D3）用于次要信息
- **强调色**：柔和的橙色（#FFB347）用于按钮和解锁特效
- **文字色**：深灰（#333333）而非纯黑，减少视觉疲劳

**字体选择**：

- **标题**：思源黑体 / 阿里巴巴普惠体（免费商用，现代感）
- **正文**：苹方 / PingFang SC（系统自带，阅读性好）
- **装饰性文字**（如词云）**：手写体（如站酷快乐体），但要克制使用

**按钮设计**：

- **主按钮**："说只鸟"采用圆角矩形（圆角半径 20px），填充柔和的橙色
- **次要按钮**：描边按钮，不填充
- **悬浮按钮**：底部居中，带轻微阴影（模糊半径 10px，透明度 20%）

**卡片设计**：

- **日记卡片**：白色背景 + 极浅的阴影，圆角 12px
- **图鉴卡片**：已解锁用白色背景，未解锁用半透明灰色蒙版

------

### 4. 动画与交互

**动画原则**：缓动 > 线性，短促 > 拖沓

**鸟类飞入动画**：

- **时长**：0.8-1.2 秒
- **效果**：淡入（opacity 0→1）+ 轻微上浮（translateY +20px → 0）
- **缓动函数**：ease-out（先快后慢，符合自然物理）

**解锁特效**：

- **时长**：1.5 秒
- **效果**：鸟类周围出现光圈扩散（scale 0.8→1.5，opacity 1→0）+ 轻微震动
- **颜色**：柔和的金色（#FFD700，透明度 50%）

**页面切换**：

- **时长**：0.3 秒
- **效果**：淡入淡出，避免滑动或翻转（保持安静感）

------

### 5. 分享图设计

这是社交传播的关键，必须做到"一眼抓人"。

**布局结构**：

```
┌─────────────────────────┐
│   我的情绪天空·第 X 天    │  ← 顶部标题（手写体，小字）
├─────────────────────────┤
│                         │
│    [天空背景 + 鸟类]      │  ← 主视觉区（占 70%）
│                         │
│   关键词1  关键词2       │  ← 词云（手写体，散落分布）
│      关键词3            │
├─────────────────────────┤
│   [小程序码]  说只鸟     │  ← 底部信息栏（占 10%）
└─────────────────────────┘
```

**关键设计要点**：

- **词云字体大小**：根据词频动态调整（12-20px），最多 5 个词
- **词云位置**：随机散落在云层边缘，避免遮挡鸟类
- **小程序码**：右下角，尺寸约 80×80px，不要太大
- **整体比例**：9:16（适配小红书、朋友圈）

------

### 6. 情绪反馈的视觉语言

**成功状态**：

- 鸟类飞入 + 轻微弹跳动画
- 配合柔和的提示音（如风铃声）

**解锁新鸟**：

- 金色光圈扩散 + "新鸟类解锁"文字提示
- 图鉴进度条数字跳动（+1 动画）

**错误状态**：

- 不要用红色，改用暖灰色提示
- 文案温柔："网络开小差了，稍后再试吧"

------

### 7. 最容易踩的坑

❌ **过度设计**：加太多装饰元素（如花边、光效、粒子），破坏安静感
 ✅ **克制留白**：让天空和鸟类成为唯一焦点

❌ **色彩过于鲜艳**：高饱和度会让用户视觉疲劳
 ✅ **降低饱和度**：所有颜色饱和度控制在 30-60%

❌ **鸟类风格不统一**：有的写实有的卡通
 ✅ **统一风格**：使用同一套 AI 绘图 Prompt 批量生成

❌ **动画过长**：每个动画都 2-3 秒
 ✅ **快速反馈**：核心动画控制在 1 秒内

------

### 8. 参考产品（视觉风格）

**推荐研究**：

1. **Forest（专注森林）**：极简自然风，克制的动画
2. **Calm（冥想 App）**：治愈系配色，呼吸感强
3. **Monument Valley**：几何化但有温度的插画风格
4. **Duolingo**：Q 版但不幼稚的角色设计

**不要参考**：

- ❌ 游戏化过重的产品（如 Habitica）
- ❌ 过度扁平化的产品（如早期 iOS 7 风格）
- ❌ 赛博朋克 / 暗黑风格（与治愈定位相悖）

------

## 总结：三个核心原则

1. **自然 > 人工**：色彩、动画都要模拟自然界的节奏
2. **克制 > 炫技**：少即是多，留白比填充更重要
3. **温度 > 冰冷**：即使是极简风格，也要保留情感温度

这套美术风格的目标是：让用户感觉自己在一个安静的自然空间里，而不是在一个热闹的社交产品里。
