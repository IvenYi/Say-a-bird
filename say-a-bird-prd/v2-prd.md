## 1. 项目概述

### 1.1 一句话描述

> 一款通过情绪日记生成鸟类插画、积累个人专属"情绪天空"的微信小程序——你的自然系情绪树洞。

### 1.2 项目背景

**为什么要做**：

当代年轻人面临日益增长的情绪压力，但传统的情绪日记产品存在两个核心问题：一是"写日记"门槛高、难以坚持；二是纯文字记录缺乏情感反馈，用户感受不到被倾听。市场上已有的情绪可视化产品（如 Mood Chonk 的情绪小怪兽、MoodFlora 的情绪花园）验证了"情绪→视觉化生物"这一模式的用户需求，但尚无产品将鸟类这一自然元素与情绪记录结合，更没有"情绪天空"这种具有时间积累感的可视化形式。

同时，中国观鸟群体正在快速增长，观鸟社群活跃度高但缺乏数字化的情感连接工具。这为产品提供了精准的冷启动渠道和天然的种子用户群。

**当前现状**：

用户目前的情绪记录方式主要有：手动写日记（门槛高、难坚持）、使用通用情绪追踪 App（如 Daylio、Moodistory，功能实用但缺乏情感温度）、在社交平台发泄（缺乏隐私性）、或者干脆不记录。市场上"情绪+可视化生物"赛道的产品（Mood Chonk、Voidpet Garden）均为海外产品，中国市场存在空白。

**核心假设**：

1. 用户愿意通过"说话/打字"的低门槛方式记录情绪（而非手动选择情绪标签）
2. "情绪→鸟类"的映射能为鸟类爱好者和泛自然兴趣用户提供足够的情感共鸣和收集动力
3. "情绪天空"的视觉积累效果具有足够的社交传播力（小红书/朋友圈分享）
4. 观鸟社群可以作为有效的冷启动渠道

### 1.3 项目目标

| 目标类型 | 描述 | 衡量指标 | 目标值 |
|---------|------|---------|--------|
| 核心目标 | 验证"情绪日记→鸟类生成"核心玩法的用户接受度 | 次日留存率 | MVP阶段 ≥ 30% |
| 核心目标 | 建立种子用户群 | 月活跃用户数（MAU） | 上线3个月 ≥ 5,000 |

| 非目标 | 不做心理咨询/诊断功能 | — | — |
| 非目标 | 不做实时社交聊天功能 | — | — |

## 2. 目标用户

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| 核心用户群 | 两类并行：(A) 18-35岁有情绪记录需求的年轻人，追求治愈感和仪式感；(B) 观鸟爱好者，对鸟类有天然认同感 |
| 用户规模估算 | 中国观鸟爱好者约200-500万；泛自然/治愈兴趣年轻人群体数千万级 |
| 使用频率 | 日活场景：每天1-3次情绪记录；周活场景：回顾情绪天空、查看图鉴、分享 |
| 技术水平 | 微信小程序熟练用户，无需额外学习成本 |
| 核心诉求 | (A) 低门槛的情绪倾诉出口 + 有温度的情感反馈 + 可积累的视觉成就感；(B) 鸟类知识融入日常 + 独特的鸟类收集体验 |

### 2.2 用户分层

- **P0 用户A**：有情绪记录习惯或意愿的年轻人（18-35岁），追求治愈感和仪式感 — 必须满足
- **P0 用户B**：观鸟爱好者，对鸟类品种有认知，会关注鸟类映射的准确性和知识性 — 必须满足（冷启动核心用户）
- **P1 用户**：纯粹被视觉效果吸引的泛用户，可能从小红书/朋友圈看到分享后进入 — 应该满足

## 3. 核心场景

### 场景 0：首次使用引导（Onboarding）

- **触发条件**：用户首次打开小程序
- **用户行为**：
  1. 展示欢迎页：一句话介绍产品概念（"说出你的心情，让它变成一只鸟"）+ 治愈风格的插画
  2. 引导用户完成第一次情绪记录（预设引导文案："今天过得怎么样？随便说说吧"）
  3. 系统识别情绪，展示第一只鸟的生成过程（放慢动画，增加仪式感）
  4. 鸟类飞入空白天空，配合文字说明："这是你的第一只鸟——{鸟类名}，它代表你此刻的{情绪}"
  5. 简要提示：天空会随时间积累更多鸟类，每月鸟类会迁徙到记忆森林
- **期望结果**：用户在30秒内理解产品玩法，完成第一次记录，感受到"哇"的惊喜
- **异常处理**：用户跳过引导时，首页天空显示一只示例鸟（半透明）+ 引导气泡

### 场景 1：日常情绪倾诉

- **触发条件**：用户在一天中经历了某种情绪波动（开心、烦躁、焦虑、感动等），想要倾诉
- **用户行为**：
  1. 打开小程序，进入首页（当前天空画布）
  2. 点击"说只鸟"按钮，进入输入界面
  3. 通过文字输入或语音输入，自由描述今天发生的事和感受
  4. 系统自动识别情绪，展示匹配的鸟类预览（品种+姿势+色调）
  5. 用户可选择确认或手动调整情绪标签
  6. 确认后，鸟类以动画形式飞入天空画布
- **期望结果**：用户感受到"被倾听"，看到自己的情绪化身为一只具体的鸟，获得情感满足。记录次数不受限制，随时可以倾诉
- **异常处理**：情绪识别不准确时，允许用户手动调整情绪标签（从预设列表中选择）；网络异常时用户可继续输入文字（语音不可用），内容本地缓存，界面显示"等待网络连接后生成你的鸟"占位状态，恢复网络后自动调用NLP并生成鸟类，以通知形式告知用户"你的鸟已经飞到天空了"

### 场景 2：回顾情绪天空与记忆森林

- **触发条件**：用户想回顾自己的情绪历程，或者单纯想看看自己的天空
- **用户行为**：
  1. 打开小程序，首页即为当月天空画布（背景光照随真实时间变化，装饰元素随鸟类数量进化）
  2. 默认显示近7天的鸟类，可切换为"本月全部"
  3. 点击天空中的任意鸟类，弹出该条日记的摘要和情绪标签
  4. 点击展开可查看完整日记内容，支持编辑和删除
  5. 切换到"记忆森林"入口，浏览往月迁徙的鸟类（按月份组织的森林）
  6. 在记忆森林中同样可以点击鸟类查看历史日记
- **期望结果**：用户获得情绪回顾的满足感，感受到时间积累带来的成就感
- **异常处理**：新用户天空为空时，展示引导气泡指向"说只鸟"按钮

### 场景 3：分享情绪天空

- **触发条件**：用户觉得自己的天空很好看，或者想在社交平台展示
- **用户行为**：
  1. 在天空画布页面点击"分享"按钮
  2. 系统生成精美的天空截图（包含天空背景+所有鸟类+小程序码，不含日记内容）
  3. 用户选择分享到朋友圈、微信好友或保存到相册
- **期望结果**：生成的图片足够精美，能在社交平台引发关注和讨论；其他人扫码可直接进入小程序
- **异常处理**：鸟类过少时（<3只），提示用户"再记录几次情绪，天空会更丰富哦"

### 场景 4：鸟类图鉴收集

- **触发条件**：用户想查看自己已解锁的鸟类品种，或者好奇还有哪些鸟类未解锁
- **用户行为**：
  1. 进入"图鉴"页面，查看已解锁和未解锁的鸟类
  2. 已解锁的鸟类展示品种名称、鸟类小知识、对应情绪、首次出现日期
  3. 未解锁的鸟类显示为剪影+解锁提示（如"当你感到宁静时，它会出现"）
  4. 季节限定鸟类有特殊标识和倒计时
- **期望结果**：激发用户的收集欲望，增加使用频率和留存；观鸟爱好者能从图鉴中获得鸟类知识的满足感
- **异常处理**：季节限定鸟类过期后，已解锁的保留，未解锁的标注"已迁徙，等待下一季"

### 4.2 功能详述

#### 功能：情绪输入与识别

- **目的**：让用户以最低门槛倾诉情绪，系统自动理解并给出反馈
- **输入**：用户的文字或语音（语音先通过微信语音识别API转为文字）
- **处理逻辑**：
  1. 文本经 NLP 情绪分析，输出情绪维度（快乐、悲伤、愤怒、焦虑、平静、感动、兴奋、孤独等）和强度（1-5级）
  2. 支持复合情绪识别（如"开心但有点累"→ 快乐3级 + 疲惫2级，取主情绪）
  3. 情绪分析推荐接入腾讯云NLP情感分析API（与微信生态集成度高，中文支持好），备选百度NLP
- **输出**：情绪标签 + 情绪强度 + 匹配的鸟类预览
- **边界条件**：
  - 输入内容过短（<5字）时提示用户多说一些
  - 输入内容无明显情绪倾向时默认为"平静"
  - 敏感内容（自伤/自杀倾向关键词）触发关怀提示："看起来你现在可能需要一些支持"，并展示心理援助热线（全国24小时心理危机干预热线：400-161-9995）
  - 用户对识别结果不满意时，可从预设情绪列表中手动选择（8种基础情绪+扩展情绪）
  - 单日记录上限：免费用户20次/天，VIP用户50次/天（覆盖99.9%正常使用场景）。触达上限时提示"今天已经说了很多了，好好休息，明天继续吧"（保持温暖语气，不产生付费压力）
- **交互说明**：输入界面简洁温暖，底部为文字输入框+语音按钮，上方实时显示情绪识别结果的柔和动画

#### 功能：情绪→鸟类映射引擎

- **目的**：将抽象的情绪转化为具体的鸟类形象，建立情感连接
- **处理逻辑**：三维映射模型
  1. **品种维度**（~50种）：每种鸟类对应一种核心情绪隐喻（MVP首批20种见附录10.4）
  2. **姿势维度**（每种3-4种）：对应情绪的表达方式
     - 展翅 → 释放/兴奋
     - 栖息 → 平静/休憩
     - 飞翔 → 自由/向往
     - 缩头 → 低落/退缩
  3. **色调维度**（每种3-4种）：对应情绪的色彩温度
     - 暖色调（橙/黄/红）→ 积极/热烈
     - 冷色调（蓝/紫/灰）→ 忧郁/沉思
     - 柔和色调（粉/浅绿/米白）→ 平和/温柔
     - 鲜艳色调（翠绿/亮蓝/金色）→ 活力/希望
- **素材规模**：MVP 20种×3姿势×3色调=180张；完整版 50种×4姿势×4色调=800张
- **输出**：一只具体的鸟类插画（品种+姿势+色调的唯一组合）
- **边界条件**：同一天内相同情绪不重复生成完全相同的鸟（在同品种内随机选择不同姿势/色调变体）
- **鸟类知识融入**：每只鸟生成时附带一句鸟类小知识（如"你知道吗？蜂鸟每秒振翅可达80次"），满足观鸟爱好者的知识需求

#### 功能：天空画布（实时光照+进化装饰 双层系统）

- **目的**：为用户提供沉浸式的情绪可视化空间
- **双层架构**：
  - **底层：实时光照**（根据用户设备本地时间自动变化，不可控制）
    - 06:00-08:00 清晨（柔和的橙粉色渐变光照）
    - 08:00-17:00 白天（明亮蓝色光照）
    - 17:00-19:00 黄昏（金色暖光照）
    - 19:00-22:00 傍晚（深蓝渐变光照）
    - 22:00-06:00 夜晚（深色光照，概率5%出现极光光效彩蛋）
  - **上层：进化装饰**（根据当月鸟类数量叠加装饰元素）
    - 1-5只鸟：基础天空（少量云朵）
    - 6-15只鸟：丰富云层 + 远处山峦轮廓
    - 16-30只鸟：云层中出现彩虹碎片 / 夜晚出现星座连线
    - 30只以上：完整彩虹 / 流星雨 / 极光常驻（终极状态）
  - 两层叠加渲染：光照决定整体明暗和色温，装饰元素在光照之上叠加显示
- **鸟类展示**：
  - 默认显示近7天的鸟类（活跃鸟类），更早的鸟类缩小为远景小图标
  - 用户可切换"本月全部"视图查看所有当月鸟类
  - 鸟类在画布上的位置由系统自动布局，避免重叠，模拟自然栖息状态
  - 支持手势缩放和拖动浏览
- **交互说明**：首页即为天空画布，底部浮动"说只鸟"按钮，顶部为日期切换和记忆森林入口

#### 功能：记忆森林与鸟类迁徙

- **目的**：解决天空鸟类过多的问题，同时创造有温度的情绪归档体验
- **处理逻辑**：
  1. 每月最后一天 23:59，当月天空中的所有鸟类自动"迁徙"到该月的记忆森林
  2. 迁徙时播放一段温馨的鸟群飞行动画（用户下次打开时触发）
  3. 新月份的天空重新开始，空白画布
  4. 记忆森林以月份为单位组织，每个月是一片独立的森林场景
  5. 森林场景的丰茂程度与该月鸟类数量正相关（少量鸟→稀疏树林，大量鸟→茂密森林）
- **输出**：按月份排列的森林列表，每片森林中可浏览该月所有鸟类和日记
- **交互说明**：记忆森林入口在首页顶部，进入后为横向滚动的月份时间线，点击月份进入对应森林

#### 功能：日记编辑与删除

- **目的**：让用户对已记录的内容有完整的控制权
- **编辑**：用户可修改日记文字内容；修改后可选择是否重新进行情绪识别（如果情绪变了，对应鸟类也会更新）
- **删除**：删除日记时需二次确认（"删除后，这只{鸟类名}将从天空中消失，确定吗？"）；删除后鸟类从天空/森林中移除，图鉴中已解锁的品种不受影响
- **边界条件**：批量删除不支持（防止误操作）；已分享的天空截图不受影响（截图是静态图片）

#### 功能：通知与召回策略

- **目的**：通过温和的提醒帮助用户养成记录习惯，提升留存
- **通知类型**：
  1. **每日记录提醒**：用户可在设置中开启并选择提醒时间（默认关闭，尊重用户选择）。文案风格温暖不打扰，如"今天的天空在等你说只鸟"
  2. **周报/月报通知**：情绪报告生成后推送，如"你的本周情绪报告已生成，来看看天空里都有谁"
  3. **连续记录激励**：连续记录3/7/14/30天时推送鼓励，如"连续记录7天！你的天空越来越热闹了"
  4. **季节限定提醒**：新季节限定鸟类上线时通知
- **技术实现**：使用微信小程序订阅消息机制，需用户主动授权。首次使用引导中引导用户订阅
- **频率控制**：每日最多1条推送，避免打扰

### 5.2 差异化策略

**我们的独特价值**：

1. **鸟类主题的自然系定位**：市场上情绪可视化产品多用怪兽、植物、花朵，鸟类是空白。鸟类天然具有"自由""迁徙""歌唱"等情感隐喻，与情绪表达高度契合
2. **情绪天空+记忆森林的双空间叙事**：天空是当下，森林是回忆，迁徙是时间的流逝——这种叙事在竞品中独一无二
3. **实时光照天空**：天空随真实时间变化光照，增强沉浸感和"此刻"的仪式感
4. **精准的种子用户渠道**：观鸟社群提供了竞品不具备的冷启动优势
5. **微信小程序生态**：无需下载，分享裂变天然适配微信社交链
6. **记录不限次**：区别于限次付费模式，保护树洞体验的纯粹性

**从竞品吸收的优秀设计**：Mood Chonk 的收集机制（图鉴系统）、Voidpet Garden 的花园养成感（天空进化）、MoodFlora 的极简交互（一键记录）

## 9. 技术约束与风险

### 9.1 技术约束

- **平台约束**：微信小程序，需遵守微信小程序开发规范和审核要求；包体大小限制（主包≤2MB，分包≤20MB）；Canvas 渲染性能受限
- **性能要求**：天空画布渲染需流畅（≥30fps），鸟类数量较多时需做性能优化（如离屏渲染、分层绘制）；近7天默认视图控制同屏鸟类数量在合理范围
- **兼容性要求**：支持 iOS 和 Android 微信客户端，适配主流屏幕尺寸
- **安全/合规要求**：用户日记内容加密存储（AES-256）；需遵守《个人信息保护法》；情绪分析不做医疗诊断声明；敏感内容检测+关怀提示机制；隐私政策需在首次使用时展示并获得同意
- **后端架构**：推荐微信云开发（CloudBase）降低运维成本，包含云函数+云数据库+云存储；NLP接口通过云函数调用腾讯云API

### 9.2 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|

| 天空画布Canvas性能瓶颈 | 中 | 中 | 默认近7天视图控制同屏鸟类数（≤21只）；使用Canvas 2D API；鸟类图片做多分辨率适配（近景高清/远景缩略） |
