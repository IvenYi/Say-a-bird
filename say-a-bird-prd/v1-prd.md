## 1. 项目概述

### 1.1 一句话描述

> 一款通过情绪日记生成鸟类插画、积累个人专属"情绪天空"的微信小程序——你的自然系情绪树洞。

### 1.2 项目背景

**为什么要做**：

当代年轻人面临日益增长的情绪压力，但传统的情绪日记产品存在两个核心问题：一是"写日记"门槛高、难以坚持；二是纯文字记录缺乏情感反馈，用户感受不到被倾听。市场上已有的情绪可视化产品（如 Mood Chonk 的情绪小怪兽、MoodFlora 的情绪花园）验证了"情绪→视觉化生物"这一模式的用户需求，但尚无产品将鸟类这一自然元素与情绪记录结合，更没有"情绪天空"这种具有时间积累感的可视化形式。

同时，中国观鸟群体正在快速增长，观鸟社群活跃度高但缺乏数字化的情感连接工具。这为产品提供了精准的冷启动渠道和天然的种子用户群。

**当前现状**：

用户目前的情绪记录方式主要有：手动写日记（门槛高、难坚持）、使用通用情绪追踪 App（如 Daylio、Moodistory，功能实用但缺乏情感温度）、在社交平台发泄（缺乏隐私性）、或者干脆不记录。市场上"情绪+可视化生物"赛道的产品（Mood Chonk、Voidpet Garden）均为海外产品，中国市场存在空白。

**核心假设**：

1. 用户愿意通过"说话/打字"的低门槛方式记录情绪（而非手动选择情绪标签）
2. "情绪→鸟类"的映射能为鸟类爱好者和泛自然兴趣用户提供足够的情感共鸣和收集动力
3. "情绪天空"的视觉积累效果具有足够的社交传播力（小红书/朋友圈分享）
4. 观鸟社群可以作为有效的冷启动渠道

## 2. 目标用户

### 2.1 用户画像

| 维度 | 描述 |
|------|------|
| 核心用户群 | 18-35岁，喜欢自然/治愈风格的年轻人，有情绪记录需求但觉得传统日记太重 |
| 种子用户 | 观鸟爱好者群体，对鸟类有天然认同感，活跃于观鸟社群 |
| 用户规模估算 | 中国观鸟爱好者约200-500万；泛自然/治愈兴趣年轻人群体数千万级 |
| 使用频率 | 日活场景：每天1-3次情绪记录；周活场景：回顾情绪天空、分享 |
| 技术水平 | 微信小程序熟练用户，无需额外学习成本 |
| 核心诉求 | 低门槛的情绪倾诉出口 + 有温度的情感反馈 + 可积累的视觉成就感 |

### 2.2 用户分层

- **P0 用户**：有情绪记录习惯或意愿的年轻人（18-35岁），追求治愈感和仪式感 — 必须满足
- **P1 用户**：观鸟爱好者，对鸟类品种有认知，会关注鸟类映射的准确性和知识性 — 应该满足
- **P2 用户**：纯粹被视觉效果吸引的泛用户，可能从小红书/朋友圈看到分享后进入 — 可以满足

## 3. 核心场景

### 场景 1：日常情绪倾诉

- **触发条件**：用户在一天中经历了某种情绪波动（开心、烦躁、焦虑、感动等），想要倾诉
- **用户行为**：
  1. 打开小程序，进入首页（当前天空画布）
  2. 点击"说只鸟"按钮，进入输入界面
  3. 通过文字输入或语音输入，自由描述今天发生的事和感受
  4. 系统自动识别情绪，展示匹配的鸟类预览（品种+姿势+色调）
  5. 用户确认后，鸟类以动画形式飞入天空画布
- **期望结果**：用户感受到"被倾听"，看到自己的情绪化身为一只具体的鸟，获得情感满足
- **异常处理**：情绪识别不准确时，允许用户手动调整情绪标签；网络异常时本地缓存输入内容，恢复后自动同步

### 场景 2：回顾情绪天空与记忆森林

- **触发条件**：用户想回顾自己的情绪历程，或者单纯想看看自己的天空
- **用户行为**：
  1. 打开小程序，首页即为当月天空画布（背景随真实时间变化：白天蓝天、傍晚晚霞、夜晚星空）
  2. 点击天空中的任意鸟类，弹出该条日记的摘要和情绪标签
  3. 点击展开可查看完整日记内容
  4. 切换到"记忆森林"入口，浏览往月迁徙的鸟类（按月份组织的森林）
  5. 在记忆森林中同样可以点击鸟类查看历史日记
- **期望结果**：用户获得情绪回顾的满足感，感受到时间积累带来的成就感
- **异常处理**：新用户天空为空时，展示引导动画和示例鸟类

### 场景 3：分享情绪天空

- **触发条件**：用户觉得自己的天空很好看，或者想在社交平台展示
- **用户行为**：
  1. 在天空画布页面点击"分享"按钮
  2. 系统生成精美的天空截图（包含天空背景+所有鸟类+小程序码，不含日记内容）
  3. 用户选择分享到朋友圈、微信好友或保存到相册
- **期望结果**：生成的图片足够精美，能在社交平台引发关注和讨论；其他人扫码可直接进入小程序
- **异常处理**：鸟类过少时（<3只），提示用户"再记录几次情绪，天空会更丰富哦"

### 场景 4：鸟类图鉴收集

- **触发条件**：用户想查看自己已解锁的鸟类品种，或者好奇还有哪些鸟类未解锁
- **用户行为**：
  1. 进入"图鉴"页面，查看已解锁和未解锁的鸟类
  2. 已解锁的鸟类展示品种名称、对应情绪、首次出现日期
  3. 未解锁的鸟类显示为剪影+解锁提示（如"当你感到宁静时，它会出现"）
  4. 季节限定鸟类有特殊标识和倒计时
- **期望结果**：激发用户的收集欲望，增加使用频率和留存
- **异常处理**：季节限定鸟类过期后，已解锁的保留，未解锁的标注"已迁徙，等待下一季"

### 4.2 功能详述

#### 功能：情绪输入与识别

- **目的**：让用户以最低门槛倾诉情绪，系统自动理解并给出反馈
- **输入**：用户的文字或语音（语音先通过微信语音识别API转为文字）
- **处理逻辑**：
  1. 文本经 NLP 情绪分析，输出情绪维度（快乐、悲伤、愤怒、焦虑、平静、感动、兴奋、孤独等）和强度（1-5级）
  2. 支持复合情绪识别（如"开心但有点累"→ 快乐3级 + 疲惫2级，取主情绪）
  3. 情绪分析可调用云端 AI 接口（如百度NLP、腾讯NLP）或自建轻量模型
- **输出**：情绪标签 + 情绪强度 + 匹配的鸟类预览
- **边界条件**：输入内容过短（<5字）时提示用户多说一些；输入内容无明显情绪倾向时默认为"平静"；敏感内容（自伤/自杀倾向关键词）触发关怀提示并推荐专业资源
- **交互说明**：输入界面简洁温暖，底部为文字输入框+语音按钮，上方实时显示情绪识别结果的柔和动画

#### 功能：情绪→鸟类映射引擎

- **目的**：将抽象的情绪转化为具体的鸟类形象，建立情感连接
- **处理逻辑**：三维映射模型
  1. **品种维度**（~50种）：每种鸟类对应一种核心情绪隐喻
     - 蜂鸟 → 焦虑/忙碌（快速振翅）
     - 猫头鹰 → 深思/忧郁（夜行、沉思）
     - 天鹅 → 优雅/感动（高贵、纯净）
     - 麻雀 → 日常/平静（最常见、朴实）
     - 鹰 → 愤怒/力量（锐利、强势）
     - 金丝雀 → 快乐/歌唱（明亮、欢快）
     - 企鹅 → 孤独/坚韧（寒冷中的坚持）
     - 凤头鹦鹉 → 兴奋/张扬（色彩丰富）
     - ...（完整映射表见附录）
  2. **姿势维度**（每种3-4种）：对应情绪的表达方式
     - 展翅 → 释放/兴奋
     - 栖息 → 平静/休憩
     - 飞翔 → 自由/向往
     - 缩头 → 低落/退缩
  3. **色调维度**（每种3-4种）：对应情绪的色彩温度
     - 暖色调（橙/黄/红）→ 积极/热烈
     - 冷色调（蓝/紫/灰）→ 忧郁/沉思
     - 柔和色调（粉/浅绿/米白）→ 平和/温柔
     - 鲜艳色调（翠绿/亮蓝/金色）→ 活力/希望
- **素材规模**：50种鸟类 × 4姿势 × 4色调 = ~800张预制插画
- **输出**：一只具体的鸟类插画（品种+姿势+色调的唯一组合）
- **边界条件**：同一天内相同情绪不重复生成完全相同的鸟（在同品种内随机选择不同姿势/色调变体）

#### 功能：天空画布与实时背景

- **目的**：为用户提供沉浸式的情绪可视化空间
- **处理逻辑**：
  1. 天空背景根据用户设备的本地时间实时变化：
     - 06:00-08:00 清晨（柔和的橙粉色渐变）
     - 08:00-17:00 白天（蓝天白云）
     - 17:00-19:00 黄昏（金色晚霞）
     - 19:00-22:00 傍晚（深蓝渐变，星星开始出现）
     - 22:00-06:00 夜晚（深色星空，概率出现极光特效）
  2. 极光为随机彩蛋，出现概率约 5%，增加惊喜感
  3. 鸟类在画布上的位置由系统自动布局，避免重叠，模拟自然栖息状态
  4. 支持手势缩放和拖动浏览
- **天空进化机制**：随着当月鸟类数量增加，天空背景会叠加进化效果
  - 1-5只鸟：基础天空
  - 6-15只鸟：天空出现更丰富的云层/星座
  - 16-30只鸟：天空出现彩虹/流星等特效
  - 30只以上：天空达到"满天繁星"或"极光常驻"等终极状态
- **交互说明**：首页即为天空画布，底部浮动"说只鸟"按钮，顶部为日期和记忆森林入口

#### 功能：记忆森林与鸟类迁徙

- **目的**：解决天空鸟类过多的问题，同时创造有温度的情绪归档体验
- **处理逻辑**：
  1. 每月最后一天 23:59，当月天空中的所有鸟类自动"迁徙"到该月的记忆森林
  2. 迁徙时播放一段温馨的鸟群飞行动画（用户下次打开时触发）
  3. 新月份的天空重新开始，空白画布
  4. 记忆森林以月份为单位组织，每个月是一片独立的森林场景
  5. 森林场景的丰茂程度与该月鸟类数量正相关（少量鸟→稀疏树林，大量鸟→茂密森林）
- **输出**：按月份排列的森林列表，每片森林中可浏览该月所有鸟类和日记
- **交互说明**：记忆森林入口在首页顶部，进入后为横向滚动的月份时间线，点击月份进入对应森林

### 5.2 差异化策略

**我们的独特价值**：

1. **鸟类主题的自然系定位**：市场上情绪可视化产品多用怪兽、植物、花朵，鸟类是空白。鸟类天然具有"自由""迁徙""歌唱"等情感隐喻，与情绪表达高度契合
2. **情绪天空+记忆森林的双空间叙事**：不只是收集，而是构建一个有时间流动感的情绪世界。天空是当下，森林是回忆，迁徙是时间的流逝——这种叙事在竞品中独一无二
3. **实时天空背景**：天空随真实时间变化，增强沉浸感和"此刻"的仪式感
4. **精准的种子用户渠道**：观鸟社群提供了竞品不具备的冷启动优势
5. **微信小程序生态**：无需下载，分享裂变天然适配微信社交链

**功能差异点**：竞品普遍缺乏"时间积累→空间进化"的机制，我们的天空进化和鸟类迁徙填补了这一空白

**体验差异点**：竞品多为"选择情绪标签"的被动记录，我们通过NLP实现"自由倾诉→自动识别"的主动倾听体验

### 6.3 数据流向

```
用户输入（文字/语音）
    ↓
语音转文字（微信API）→ 文本内容
    ↓
NLP情绪分析（云端API）→ 情绪标签 + 强度
    ↓
鸟类映射引擎 → 品种 × 姿势 × 色调 → 鸟类ID
    ↓
存储：日记记录（用户ID + 文本 + 情绪 + 鸟类ID + 时间戳）
    ↓
渲染：天空画布更新（新鸟类飞入动画）
    ↓
月末触发：鸟类迁徙 → 记忆森林归档
```

## 7. 数据指标

### 7.1 北极星指标

- **指标名称**：月活跃记录用户数（Monthly Active Recording Users）
- **计算方式**：当月至少完成1次情绪记录的独立用户数
- **目标值**：上线6个月达到 20,000 MARU

### 7.2 过程指标

| 指标 | 定义 | 采集方式 | 基线 | 目标 |
|------|------|---------|------|------|
| 次日留存率 | 次日回访用户/新增用户 | 埋点 | 待验证 | ≥ 30% |
| 7日留存率 | 7日内回访用户/新增用户 | 埋点 | 待验证 | ≥ 15% |
| 日均记录次数 | 总记录次数/DAU | 埋点 | 待验证 | ≥ 1.5次 |
| 分享率 | 分享用户数/MAU | 埋点 | 待验证 | ≥ 10% |
| 图鉴解锁率 | 平均解锁鸟类数/总鸟类数 | 后端统计 | 待验证 | 30日用户 ≥ 20% |
| 付费转化率 | 付费用户/MAU | 支付系统 | 待验证 | ≥ 3% |
| 广告观看率 | 观看广告用户/触达免费上限用户 | 广告SDK | 待验证 | ≥ 40% |

### 7.3 防御指标

| 指标 | 底线值 | 监控方式 |
|------|--------|---------|
| 情绪识别准确率 | ≥ 80%（用户未手动修正的比例） | 后端统计修正率 |
| 页面加载时间 | 首页 ≤ 2秒 | 性能监控 |
| 崩溃率 | ≤ 0.5% | 微信小程序后台 |
| 用户投诉率 | ≤ 0.1% | 客服系统 |

### 8.1 MVP 范围定义

**包含**：
- 微信授权登录
- 文字+语音情绪输入
- NLP 情绪识别（接入第三方API）
- 情绪→鸟类映射（首批20种鸟类 × 3姿势 × 3色调 ≈ 180张素材）
- 天空画布（实时时间背景 + 鸟类展示 + 点击查看日记）
- 每日2次免费 + 激励视频广告解锁
- 基础分享功能（天空截图+小程序码）

**不包含**：
- 记忆森林（MVP阶段天空不做月度迁徙，鸟类持续积累）
- 鸟类图鉴
- 天空进化机制
- VIP 订阅
- 装饰商城
- 情绪报告
- 送鸟给朋友
- 季节限定鸟类

**MVP 成功标准**：
- 上线30天内获取 1,000+ 注册用户（主要来自观鸟社群推广）
- 次日留存率 ≥ 25%
- 日均记录次数 ≥ 1.2次/活跃用户
- 用户对"情绪→鸟类"映射的满意度 ≥ 70%（通过应用内反馈收集）

## 9. 技术约束与风险

### 9.1 技术约束

- **平台约束**：微信小程序，需遵守微信小程序开发规范和审核要求；包体大小限制（主包≤2MB，分包≤20MB）；Canvas 渲染性能受限
- **性能要求**：天空画布渲染需流畅（≥30fps），鸟类数量较多时需做性能优化（如离屏渲染、分层绘制）
- **兼容性要求**：支持 iOS 和 Android 微信客户端，适配主流屏幕尺寸
- **安全/合规要求**：用户日记内容加密存储；需遵守《个人信息保护法》；情绪分析不做医疗诊断声明；敏感内容（自伤倾向）需有安全兜底机制

### 9.2 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|---------|
| 插画素材制作周期长、成本高 | 中 | 高 | 分批制作：MVP先做20种，验证后再扩展；使用AI辅助生成+人工精修的流程 |
| NLP情绪识别准确率不够 | 中 | 高 | 选择成熟的第三方API（百度NLP/腾讯NLP）；提供手动修正机制作为兜底；持续收集用户修正数据优化模型 |
| 天空画布Canvas性能瓶颈 | 中 | 中 | 限制单屏同时渲染鸟类数量（超出部分做虚拟滚动）；使用WebGL加速；鸟类图片做多分辨率适配 |
| 微信小程序审核风险 | 低 | 高 | 提前了解审核规范；情绪相关功能避免涉及医疗/心理诊断措辞；准备审核说明文档 |
| 冷启动用户获取不及预期 | 中 | 中 | 多渠道并行：观鸟社群+小红书KOC合作+朋友圈裂变；准备种子用户激励方案 |
| 用户对鸟类映射不感兴趣 | 低 | 高 | MVP阶段快速验证；准备Plan B（如果鸟类不够吸引，可扩展为更多自然生物） |
